🏈What Is NFL Squares (Private / Self-Managed)?

NFL Squares is a grid-style contest where entrants pick squares on a 10×10 grid and win based on the last digits of the score at the end of a quarter or game. In the Private (Self-Managed) product, the commissioner runs the contest using Splash's tools, but handles all entry fees and payouts themselves, outside the platform.

⭐ Key facts every rep should know

Not real money on Splash — Splash itself does not process or hold funds directly. Commissioners choose how payments work: enable Contest Vault (powered by LeagueSafe) so members can pay for their square(s) directly in-platform, with the commissioner controlling payouts, ties, and other payout rules; or skip Contest Vault and collect member square fees themselves, off-platform — in which case the commissioner covers the per-grid service fee.

Pricing is per grid — NFL Squares grids are $12.95 each (this is what the commissioner pays Splash to create/run the grid — separate from whatever the commissioner charges their own entrants). If Contest Vault is enabled, commissioners don’t pay this service fee.

Joining is invite-link only — there are no entry codes or passwords. Each contest has a unique, reusable invite link.

Scoring is currently Manual only — the commissioner marks winning squares themselves. Automated scoring (by final score or by quarter) is visible in the UI but not yet available.

Where NFL Squares fits among NFL Private formats

Splash offers several Private (Self-Managed) NFL formats. Squares is the grid-based one — good for group/office-pool style participation:

  • NFL Pick’em — pick winners of each week’s games.
  • NFL Pick’em Confidence — pick winners and assign confidence points.
  • NFL Squares — grid-style, win by score digits (this guide).

All of these are self-managed: the commissioner handles payments/payouts externally, and Splash provides the contest tooling.

🏗️The Squares Grid Tab — Creating Grids

Once inside the contest, the commissioner opens the Squares grid tab from the left navigation. Under Create Grid, there are two setup paths — the commissioner picks one and clicks Continue:

  • Single Game — assign a grid to one specific NFL game. More games/grids can always be added later.
  • Series of Games — auto-generate a grid for every game in a series (e.g., every Monday Night Football game, or every game on one team’s schedule).

3.1  Single Game Setup

If Single Game is selected, the commissioner configures the following fields:

Basic Settings

  • Week of Game — select the season week (e.g., Week 1).
  • Game — pick the specific matchup for that week.
  • Grid Placement — Standalone, or Child of parent (linked to another grid).
  • Grid Name — optional; defaults to the matchup name if left blank.
  • Max Number of Squares Per User — set a limit, or choose Unlimited.
  • Scoring Mode — Manual is currently the only available option (Automated by final score / by quarter are shown in the UI but not yet available — a common question, see FAQ).

Advanced Settings

  • Grid Size — 25, 50, or 100 squares.
  • Sets of Numbers on the Grid — 1 set (used the whole game) or 4 sets (a new set each quarter).
  • Axis Numbers on the Grid Are — Randomly Assigned (auto-populated when the commissioner locks the grid or the game starts, whichever is first) or Custom Assigned (commissioner fills them in manually).
  • Axis Labels — Away on Top/Home on Side, or Home on Top/Away on Side.
  • Number each pickable square (1–100) — toggle a reference number on each square.
  • Paid Tracking — No Tracking / Show tracking for commissioner only / Show tracking for everyone.

The commissioner clicks Create grid to finalize.

3.2  Series of Games Setup

If Series of Games is selected, the commissioner first chooses a Series Type: By Schedule or By Team.

🗓️ By Schedule

Builds a grid series around a recurring schedule slot.

  • Schedule Slot — select the recurring slot to base the series on (e.g., Monday Night Football).

Once selected, the system generates one grid per matchup in that slot — e.g., choosing Monday Night Football creates 18 grids. The first grid is marked the Parent Grid; the commissioner can expand “Show N more” to see the rest.

🏟️ By Team

Builds a grid series around one team’s full schedule.

  • Team — select the team to base the series on (e.g., Minnesota Vikings).

Once selected, the system generates one grid per game on that team’s schedule — e.g., choosing the Vikings creates 17 grids. Again, the first grid is the Parent Grid.

Series-wide settings (apply to every grid in the series)

  • Scoring Mode — Manual only (Automated options shown but not yet available).
  • Grid Size — 25, 50, or 100.
  • Sets of Numbers — 1 set, or 4 (one per quarter).
  • Axis Numbers — Randomly Assigned or Custom Assigned.
  • Axis Labels — Away-top/Home-side, or Home-top/Away-side.
  • Numbering toggle — on/off.
  • Paid Tracking — No Tracking / commissioner-only / everyone.

Clicking Create grid series generates every grid in the series at once.

SECTION 5

⚙️Edit Grid Settings (Gear Icon)

There are two ways to get to Edit Grid Settings:

  • Click the gear icon directly on a grid in the Grid View, or
  • Go to Contest Details > Contest Settings > Grids, then click Edit Settings on the grid you want to edit.

Editable Fields

  • Week of Game
  • Game
  • Grid Name
  • Max Number of Squares Per User
  • Scoring Mode (Manual only, currently)
  • Advanced Settings — click Review to expand grid size, sets of numbers, axis assignment, axis labels, numbering, and paid tracking

Grid Actions (right panel)

  • Lock Grid — reveals axis numbers and freezes claims. Irreversible once the game starts.
  • Delete Grid — permanently removes the grid; members lose access. Destructive action, shown in red.
  • Detach Grid — flips a child grid into a standalone grid; picks from the parent stop mirroring here.
  • Promote Grid — makes this grid the new series root; the current parent becomes a child and siblings re-parent here. Claims are preserved.
  • + Create New Grid — quick link to add another grid without leaving the screen.

Bottom Actions

  • Cancel — discards changes.
  • Save changes — saves updates to the grid.
🚨 CS Note

“Lock Grid” and “Delete Grid” are irreversible/destructive. “Detach” and “Promote” restructure a grid series and can’t be casually undone either. If a commissioner reaches out after any of these, set expectations clearly.

See Section 12: Grid Actions for a full plain-language breakdown of what each action means for the commissioner and for members.

SECTION 8

🔗Parent / Child Grid Behavior

In any grid series (By Schedule or By Team), only the Parent Grid accepts square selections. Once picks are made on the parent, they automatically mirror to every child grid in the series.

When a commissioner or entrant views a child grid, a modal appears: “Picks happen on the parent grid — this grid is part of a series. Pick squares on the parent grid and they’ll mirror to every child grid automatically.”

  • Stay here — dismiss the modal and continue viewing the child grid (read-only for picks).
  • Go to parent grid — navigate directly to the parent grid to make picks.
💬 Common support scenario

“I can’t pick a square on Grid 5, it’s greyed out / read-only!” — this is expected if Grid 5 is a child grid. Direct the commissioner/entrant to the Parent Grid (Grid 1, typically) to make picks; they’ll mirror automatically.

SECTION 9

✅Commissioner Square Assignment

When the commissioner clicks the + on an open square, the Commissioner Square Assignment modal appears with two ways to fill that square.

🤔 Which option should the commissioner choose?

Option 1, Assign to an Entrant, is recommended — it lets the commissioner maintain a roster with contact info, number of squares assigned, and custom notes per entrant. If someone hasn’t joined yet, the commissioner can send them the invite link.

Option 2, Assign to a Guest, is faster if the commissioner just wants to enter a name without tracking anything else (e.g., someone paying cash who won’t have an account).

Option 1 — Assign to an Entrant

  • Select Entrant — dropdown of existing entrants in the contest.
  • Click Assign to Entrant to lock that square to the selected entrant.

Option 2 — Assign to a Guest

  • Guest Name — free-text field for someone who isn’t a registered entrant.
  • Click Assign to Guest to assign the square under that name.

SECTION 12

🔒Grid Actions: Lock, Delete, Detach & Promote

📍 Where to find these

Go to Contest Details > Contest Settings > Grids, then click Edit Settings on the grid you want to act on. All four actions live in the right-hand panel of Edit Grid Settings.

🔒 Lock Grid

Reveals the axis numbers and freezes claims. Irreversible once the game starts.

  • For the commissioner — locking reveals the actual axis numbers (no more “?” placeholders) and immediately shuts off new claims. Useful for finalizing things early — e.g., the grid’s full, or the commissioner just wants numbers set before kickoff. Once the game itself starts, this becomes permanent — there’s no unlocking or re-hiding the numbers afterward.
  • For a member — they can see exactly which numbers landed on which row/column, and their squares (whatever they’d claimed by that point) are locked in. If they hadn’t finished claiming the squares they wanted, they’re out of luck once it’s locked.

🗑️ Delete Grid

Permanently removes this grid. Members will lose access.

  • For the commissioner — this wipes the grid entirely: every square claim, note, and axis assignment on it is gone for good. There’s no recovery.
  • For a member — the grid disappears from their view completely, along with any squares they’d claimed on it. Since money for private contests is handled outside Splash (or via Contest Vault), this is the kind of action a commissioner should only take after confirming with everyone involved — deleting a grid people already paid into could cause real disputes.

🔗 Detach Grid

Flips this grid into a standalone grid. Picks from the parent stop mirroring here.

  • For the commissioner — if this grid was a child in a series (auto-mirroring whatever gets picked on the parent), detaching turns it into its own independent grid. From that point forward, picks made on the old parent no longer show up here — this grid needs its own picks going forward.
  • For a member — any squares already mirrored onto this grid before detaching stay as they were at that moment (a snapshot). Going forward, this grid stops following the parent; to claim a new square here, it now works independently, not by picking on the parent.

⬆️ Promote Grid

Makes this grid the new series root. The current parent becomes a child; siblings re-parent here. Claims are preserved.

  • For the commissioner — this swaps which grid is the “root” of the series. Say Grid 1 is currently the parent and Grid 5 is a child — promoting Grid 5 makes it the new parent; Grid 1 becomes a child of Grid 5, and every other sibling grid re-parents to Grid 5 too. All existing claims stay exactly as they are — only which grid drives future picks changes. Handy if, say, the original parent’s game got rescheduled and the commissioner wants a different (more current) game’s grid to be the one live picks happen on.
  • For a member — nothing changes about squares they’ve already claimed. But if they want to make a new pick after a promotion, they need to go to the new parent grid to do it — not the old one.
🚨 CS Note

Lock and Delete are irreversible/destructive. Detach and Promote restructure a grid series and can’t be casually undone either — all four should be treated with the same care. If a commissioner reaches out after any of these, walk through what changed rather than assuming it can be reversed.

❓Frequently Asked Questions

Q: Is this a real-money game? Does Splash hold the pot?

No. Private (Self-Managed) contests are not real-money contests on Splash — Splash never processes or holds funds. All money is exchanged directly between the commissioner and their entrants, off-platform.

Q: How much does an NFL Squares grid cost?

$12.95 per grid. Each grid is billed individually — if a commissioner wants multiple grids for the same game (or a full series), each one is charged separately. If Contest Vault is enabled, commissioners don't pay this per-grid service fee.

Q: Why can't I mark a winning square myself / where's automated scoring?

Scoring Mode currently only supports Manual — the commissioner marks winners themselves. Automated scoring (by final score or by quarter) is visible in the settings UI as a preview of upcoming functionality but is not yet available.

Q: I'm on a child grid and can't pick a square — is something broken?

This is expected behavior. In a grid series, only the Parent Grid accepts picks; child grids are read-only and mirror whatever is picked on the parent. Direct the user to “Go to parent grid” to make their picks.

Q: How does someone join a contest?

Via a direct invite link from the commissioner only — there are no entry codes or passwords. The same link is contest-specific but can be reused/resent as many times as needed.

Q: A member says their invite link isn't working — what should I check?

Confirm they're using the most recent link from the commissioner, that the contest is still open (not locked/started), and that the entry deadline hasn't passed.

Q: Can the commissioner assign a square to someone who hasn't signed up?

Yes — use Option 2, “Assign to a Guest,” which just takes a free-text name. Option 1, “Assign to an Entrant,” is for people who have already joined via the invite link and is the recommended default since it tracks contact info and notes.

Q: Can a commissioner undo locking, deleting, detaching, or promoting a grid?

No — none of these are casually reversible. Locking reveals axis numbers and freezes claims permanently; deleting removes the grid (and member access to it) permanently; detaching stops a grid from mirroring its parent going forward; promoting changes which grid drives future picks (though existing claims are preserved). See Section 12 for the full breakdown, and set expectations clearly before a commissioner proceeds with any of these.

Q: What other NFL private formats exist besides Squares?

NFL Pick'em and NFL Pick'em Confidence — both self-managed, same principle of Splash providing tooling while the commissioner manages payments/payouts.

Q: What is the Contest Vault, and do commissioners have to use it?

It's optional. If enabled, members pay for their square(s) directly in-platform (powered by LeagueSafe), and the commissioner controls payouts, ties, and related decisions from the Vault's Settings. If a commissioner doesn't enable it, they collect member square fees themselves, off-platform, and are responsible for the per-grid service fee.

Q: How do I add a Super Bowl (Super Grid)?

Go to Grid Management > Add a New Grid, select Single Game, then choose Super Bowl as the week and fill in the remaining details.

Q: When do the X & Y axis numbers get assigned?

Numbers generate automatically at kickoff. If you'd rather lock them in early, go to Grid Settings > Show Numbers and Lock Grid to generate and lock the numbers ahead of time.

Q: What happens if my grid isn't full when numbers are supposed to generate?

Any squares that haven’t been claimed stay empty. If a winning combination lands on an empty square, the system scores it as an “Unclaimed Square.” There’s no built-in rule for handling this — commissioners should decide in advance (e.g., no winner for that quarter, rollover to next winning square, etc.) and communicate the rule to members before the game.

Q: I'm a new commissioner — how do I get my Squares pool set up before the season starts?

  1. Create your contest and choose Squares as the format.
  2. Set your grid price and number of grids needed.
  3. Invite members via link or email.
  4. Decide (and share) your house rules for ties/empty squares ahead of kickoff.
  5. Lock numbers early or let them auto-generate at kickoff.

Q: Can I run multiple grids in one contest?

Yes. Each additional grid is billed per-grid until you hit the paid-grid threshold for your contest, after which further grids are free (up to the max allowed per contest).
